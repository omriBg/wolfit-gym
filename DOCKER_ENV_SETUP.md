# 🔒 הוראות הגדרת משתני סביבה ל-Docker

## מה השתנה?

הסיסמה של מסד הנתונים הוסרה מהקובץ `docker-compose.yml` והועברה למשתני סביבה.

## איך להגדיר?

### שלב 1: צור קובץ `.env` בשורש הפרויקט

צור קובץ בשם `.env` בתיקייה הראשית של הפרויקט (באותה תיקייה שבה נמצא `docker-compose.yml`).

### שלב 2: הוסף את הסיסמה

פתח את הקובץ `.env` והוסף את השורה הבאה:

```bash
DB_PASSWORD=הסיסמה_החזקה_שלך_כאן
```

**דוגמה:**
```bash
DB_PASSWORD=MySecurePassword123!
```

### שלב 3: הרץ את Docker Compose

```bash
docker-compose up
```

## ⚠️ חשוב מאוד!

1. **לעולם אל תעלה את קובץ `.env` לגיט!**
   - הקובץ כבר מופיע ב-`.gitignore`
   - ודא שהוא לא נשמר ב-Git

2. **השתמש בסיסמה חזקה:**
   - לפחות 12 תווים
   - שילוב של אותיות, מספרים ותווים מיוחדים
   - לא להשתמש בסיסמאות ברירות מחדל

3. **בסביבת ייצור:**
   - השתמש בסיסמה שונה לחלוטין
   - אל תשתמש בסיסמה הזהה לסביבת פיתוח

## בדיקה שהכל עובד

לאחר הרצת `docker-compose up`, בדוק שהכל עובד:

```bash
# בדוק שהקונטיינרים רצים
docker-compose ps

# בדוק את הלוגים
docker-compose logs postgres
docker-compose logs backend
```

## בעיות נפוצות

### שגיאה: "DB_PASSWORD is not set"
**פתרון:** ודא שיצרת קובץ `.env` עם המשתנה `DB_PASSWORD`

### שגיאה: "Cannot connect to database"
**פתרון:** ודא שהסיסמה ב-`.env` תואמת לסיסמה במסד הנתונים

## סיכום

✅ **לפני:** הסיסמה הייתה חשופה בקוד  
✅ **אחרי:** הסיסמה נשמרת בקובץ `.env` שלא נשמר ב-Git

זה הרבה יותר בטוח! 🔒
